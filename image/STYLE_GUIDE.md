# 프로젝트 이미지 스타일 가이드

> 버전: 1.0  
> 작성일: 2026년 6월 30일  
> 적용 범위: 프로젝트에서 새로 생성하거나 편집하는 모든 이미지

## 1. 기본 방향

프로젝트 이미지는 **현대적인 스포츠 에디토리얼** 스타일을 기본으로 한다. 경기의 속도감과 현장감은 살리되, 자료조사 결과와 데이터가 명확하게 전달되는 차분한 시각 체계를 유지한다.

핵심 인상:

- 역동적이지만 과도하게 자극적이지 않음
- 전문적이고 신뢰할 수 있음
- 한국어 콘텐츠에 자연스럽게 어울림
- 모바일과 발표 자료에서 빠르게 이해할 수 있음
- 여러 이미지를 함께 배치해도 하나의 시리즈처럼 보임

## 2. 시각 스타일

### 사진·일러스트

- 경기장의 조명, 관중, 선수의 움직임에서 현장감을 표현한다.
- 사실적인 스포츠 사진 또는 정교한 디지털 에디토리얼 일러스트를 사용한다.
- 순간 동작은 선명하게, 배경은 약한 모션 블러나 심도 표현으로 분리한다.
- 인물은 자연스러운 신체 비율과 실제 경기에서 가능한 자세로 표현한다.
- 이미지 안에 불필요한 글자나 가상의 로고를 생성하지 않는다.

### 그래픽

- 넓은 여백, 명확한 정렬, 제한된 색상을 사용한다.
- 사선, 속도선, 경기장 라인 등의 요소로 운동감을 가볍게 더한다.
- 장식보다 정보의 위계를 우선한다.
- 그림자와 광택 효과는 약하게 사용하고 과도한 3D 효과는 피한다.

## 3. 색상

| 용도 | 색상 | HEX |
|---|---|---|
| 주 배경 | 딥 네이비 | `#071426` |
| 보조 배경 | 슬레이트 블루 | `#132A44` |
| 핵심 강조 | 에너지 레드 | `#E63946` |
| 데이터 강조 | 터프 그린 | `#20C997` |
| 본문·밝은 배경 | 오프화이트 | `#F5F7FA` |
| 보조 정보 | 쿨 그레이 | `#AAB4C3` |

사용 원칙:

- 한 이미지에서 주조색 1개, 강조색 1~2개만 사용한다.
- 대한민국 관련 강조에는 레드와 네이비를 사용하되 국기 문양을 장식처럼 남용하지 않는다.
- 멕시코 등 상대 팀은 해당 국가를 연상시키는 색을 보조적으로만 사용한다.
- 텍스트와 배경은 충분한 명도 대비를 확보한다.

## 4. 타이포그래피

- 한국어: `Pretendard` 또는 `Noto Sans KR`
- 영문·숫자: `Montserrat` 또는 `Inter`
- 제목은 굵고 짧게, 본문은 보통 굵기로 사용한다.
- 한 이미지에 글꼴은 최대 2종, 굵기는 최대 3단계로 제한한다.
- AI 이미지 생성 단계에서는 글자를 넣지 않고, 필요한 문구는 후편집으로 추가하는 것을 원칙으로 한다.

## 5. 구도

- 핵심 피사체는 중앙 또는 3분할 교차점에 배치한다.
- 제목이나 통계를 넣을 예정인 방향에 25~35%의 여백을 남긴다.
- 대결 구도는 좌우 균형을 사용하되, 모든 이미지를 정면 대칭으로 만들지는 않는다.
- 인포그래픽은 위에서 아래 또는 왼쪽에서 오른쪽으로 읽히도록 구성한다.
- 썸네일에서는 작은 크기로 보더라도 핵심 피사체와 대비가 유지되어야 한다.

## 6. 이미지 유형별 규격

| 용도 | 권장 비율 | 권장 크기 |
|---|---:|---:|
| 문서 대표 이미지 | 16:9 | 1600×900 |
| 발표 자료 | 16:9 | 1920×1080 |
| 카드 뉴스 | 1:1 | 1080×1080 |
| 세로형 콘텐츠 | 4:5 | 1080×1350 |
| 배너·헤더 | 3:1 | 1800×600 |
| 투명 배경 요소 | 자유 | 긴 변 1600px 이상 |

완성본은 기본적으로 PNG 또는 WebP로 저장한다. 사진 중심이고 용량을 줄여야 할 때만 고품질 JPG를 사용한다.

## 7. 생성 프롬프트 작성법

아래 구조를 기본 템플릿으로 사용한다.

```text
[이미지 목적], [주요 장면과 피사체], modern sports editorial style,
realistic stadium lighting, dynamic but credible action,
deep navy base with restrained red and green accents,
clean composition, strong visual hierarchy,
[텍스트를 넣을 위치]에 넉넉한 negative space,
[화면 비율], no text, no logo, no watermark
```

예시:

```text
대한민국과 멕시코의 국제 축구 경기 프리뷰용 대표 이미지,
서로 마주 보는 두 팀의 축구 선수를 실제 인물이 아닌 일반적인 선수로 표현,
야간 경기장과 긴장감 있는 관중석, modern sports editorial style,
realistic stadium lighting, dynamic but credible action,
deep navy base with restrained red and green accents,
clean composition, strong visual hierarchy,
왼쪽 상단에 제목을 넣을 넉넉한 negative space,
16:9, no text, no official crest, no logo, no watermark
```

## 8. 피해야 할 요소

- 공식 FIFA·대회·협회·구단 로고를 허가 없이 재현하는 것
- 실제 선수와 혼동될 정도로 특정 인물의 얼굴을 부정확하게 생성하는 것
- 읽을 수 없는 AI 생성 문자, 유니폼의 가짜 후원사와 가짜 엠블럼
- 손가락, 팔다리, 축구공 크기 등 신체·사물 비율 오류
- 지나친 네온 색상, 불꽃, 번개, 렌즈 플레어
- 과도한 국기 사용과 국가에 대한 고정관념적 표현
- 출처가 불분명한 사진이나 저작권이 있는 이미지를 그대로 변형하는 것

## 9. 파일 관리 규칙

파일명은 영문 소문자와 하이픈을 사용한다.

```text
주제-용도-버전.확장자
```

예시:

```text
korea-mexico-match-preview-v01.png
korea-mexico-match-stats-v01.webp
project-header-v02.jpg
```

권장 하위 폴더:

```text
image/
├─ generated/   # AI로 생성한 원본
├─ edited/      # 후편집한 작업본
├─ final/       # 실제 프로젝트에 사용하는 최종본
└─ reference/   # 사용 권한과 출처를 확인한 참고 이미지
```

같은 이미지의 수정본은 `v01`, `v02`처럼 버전을 올린다. 최종본에도 버전을 유지하고 `final-final` 같은 이름은 사용하지 않는다.

## 10. 생성 전 확인표

- [ ] 이미지의 용도와 출력 비율이 정해졌는가?
- [ ] 기존 이미지와 색상·조명·구도가 일관적인가?
- [ ] 글자를 넣을 여백이 충분한가?
- [ ] 공식 로고와 상표를 무단으로 포함하지 않았는가?
- [ ] 인물과 축구 장면이 자연스러운가?
- [ ] 파일명이 관리 규칙을 따르는가?
- [ ] 최종 사용본과 생성 원본이 구분되어 있는가?

## 11. 가이드 갱신

현재 프로젝트 소개 문서의 세부 목표와 기능은 작성 중이므로, 프로젝트의 최종 매체와 브랜드 방향이 정해지면 다음 항목을 보완한다.

- 공식 로고와 브랜드 색상
- 고정 타이포그래피
- 이미지별 실제 출력 규격
- 사진과 AI 생성 이미지의 사용 범위
- 저작권 및 출처 표기 방식
